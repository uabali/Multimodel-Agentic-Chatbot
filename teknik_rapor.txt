2. AGENT STATE — src/agent/state.py
────────────────────────────────────────────────────────────────────────────────

LangGraph'ın paylaşımlı durum nesnesi. TypedDict olarak tanımlanmıştır; her
alan tek bir sorumluluğa sahiptir.

  class AgentState(TypedDict):
      messages         : Annotated[Sequence[BaseMessage], add_messages]
      documents        : list[Document]       # retriever çıktısı
      original_question: str                  # rewriter dokunmaz (web search kullanır)
      question         : str                  # rewriter güncelleyebilir
      generation       : str                  # nihai LLM yanıtı
      route            : str                  # "rag"|"direct"|"vision"|...
      relevance        : str                  # "yes"|"no"
      grader_reason    : str                  # ""|"irrelevant"|"needs_live_data"
      source_filter    : str                  # tek dosya filtreleme
      session_uploads  : list[str]            # oturum dosya listesi (follow-up RAG)
      image_data       : list[dict]           # base64 görseller
      input_type       : str                  # "text"|"image"|"audio"
      vision_context   : str                  # vision_rag_node çıktısı
      temperature      : float                # per-session override
      max_tokens       : int
      retrieval_strategy: str
      use_rerank       : bool

Kritik tasarım kararı — original_question / question ayrımı:
  Rewriter, retrieval kalitesi için soruyu yeniden formüle edebilir. Ancak
  web_search_node, doğal dil kalıplarına daha iyi yanıt verdiğinden
  original_question'ı kullanır. Bu iki alanlı tasarım her iki bağlamı da
  optimize eder.

grader_reason alanı (yeni):
  Grader LLM artık "no" kararında sebep de üretiyor. "needs_live_data" sebebi,
  source_filter aktifken dahi web fallback'in tetiklenmesini sağlar. Önceki
  tasarımda, yüklü belgede canlı veri gerektiren sorgularda (ör. fatura +
  güncel döviz kuru) web search hiç devreye girmiyordu.

3. GRAPH TOPOLOJİSİ — src/agent/graph.py
────────────────────────────────────────────────────────────────────────────────

LangGraph StateGraph, DAG (directed acyclic graph) olarak derlenir.

  START
    └─ router_node
         ├─ "rag"           → rewriter_node
         │                       └─ retriever_node
         │                             └─ grader_node
         │                                  ├─ "sufficient" → generator_node → END
         │                                  └─ "insufficient" → web_search_node
         │                                                         └─ generator_node → END
         ├─ "direct"        → direct_response_node → END
         ├─ "vision"        → vision_node → END
         ├─ "vision_rag"    → vision_rag_node → rewriter_node (RAG yoluna devam)
         └─ "vision_search" → vision_search_node → generator_node → END

Döngü önlemi (yapısal):
  web_search_node → generator_node kenarı grader_node'a geri dönmüyor.
  Sonsuz CRAG döngüsü runtime sayaç değil, graph topolojisi tarafından
  önleniyor. Bu OCP uyumlu bir tasarım kararıdır.

_route_decision koşullu kenarı:
  route == "vision" ise ikincil kararlar devreye girer:
    source_filter var  → "vision_rag"  (aynı turda yüklenen belge + görsel)
    is_web_query()     → "vision_search"
    diğer              → "vision"  (saf analiz, END)

  Not: session_uploads varlığı vision_rag'ı tetiklemez. Önceki oturum
  belgeleri görsel analiziyle birleştirilmemeli; yalnızca aynı turda yüklenen
  belge (source_filter) bu yola yönlendirir.

_grader_decision koşullu kenarı:
  relevance == "yes"                               → "sufficient"
  relevance == "no" AND source_filter AND
    grader_reason == "needs_live_data"             → "insufficient" (web arama)
  relevance == "no" AND source_filter (diğer)     → "sufficient"   ("belgede yok" der)
  relevance == "no" AND source_filter yok          → "insufficient" (web arama)

astream_agent() — semantic cache sarmalayıcısı:
  Grafı çalıştırmadan önce SemanticCache.lookup() çağrılır. Cache hit olursa
  graph atlanır, mock event'lar yield edilir. Cache miss sonrası graph çalışır
  ve SemanticCache.store() ile yanıt kaydedilir.
  Bağlam anahtarı: SHA-256(source_filter + sorted(session_uploads) +
  retrieval_strategy)[:16] — farklı belge setlerinde aynı soru yanlış cache
  dönmez.

Thread safety notu:
  get_graph() senkrondur (await noktası yok). asyncio cooperative scheduling
  altında iki coroutine if/set arasına giremez. Lock gereksizdir.

4. ROUTER NODE — 4 Kademe
────────────────────────────────────────────────────────────────────────────────

router_node'un karar sırası:

  Kademe 0 (0 ms)  : image_data doluysa → "vision"
  Kademe 1 (0 ms)  : source_filter varsa → "rag" (deterministik)
  Kademe 2 (0 ms)  : keyword_route(question, has_uploads) eşleşirse
  Kademe 3 (0 ms)  : session_uploads var + is_web_query() → "direct"
                     session_uploads var ama web değil  → "rag"
  Kademe 4 (~200ms): LLM fallback (max_tokens=64, temp=0.0, JSON parse)

  NOT: input_type=="audio" için önceki early return kaldırıldı. Sesli
  sorgular artık yazılı sorgularla özdeş routing mantığından geçiyor.
  Bu, sesli belge follow-up sorgularının RAG'a yönelmesini sağlar.

keyword_route() — src/agent/routing.py:
  6 regex pattern seti, sıfır dış bağımlılık:

  _DOCUMENT_PRONOUN_PATTERNS  → "bu/şu cv/dosya", "kime ait", "bu belgedeki..."
  _GENERAL_KNOWLEDGE_PATTERNS → "X nedir?" (^ anchor — uzun sorularda FP önler)
  _RAG_PATTERNS               → "belgede", "dosyaya göre", "yüklediğim..."
  _DIRECT_PATTERNS            → selamlama, matematik, hava, borsa, sohbet
  _WEB_PATTERNS               → gerçek zamanlı sinyal sözcükler
  _MCP_PATTERNS               → github, calendar, email komutları

  has_uploads=True olduğunda _GENERAL_KNOWLEDGE_PATTERNS atlanır:
  "transformer nedir?" → belge yüklüyse rag'a yönlenebilir.

  ^ anchor kritik: "bağlantısı nedir?" gibi uzun sorgular _GENERAL_KNOWLEDGE
  eşleşmesinden kaçarak doğru route alır.

5. RETRIEVER PIPELINE — src/rag/retriever.py + vectorstore.py
────────────────────────────────────────────────────────────────────────────────

5.1 HybridVectorStore (vectorstore.py)

  Qdrant üzerinde iki vektör kanalı:
    Dense  : BAAI/bge-m3, 1024 boyut, cosine benzerliği
    Sparse : FastEmbedSparse (BM25), token-level
  RetrievalMode.HYBRID (varsayılan) — fusion skoru döner.

  Dense Gate:
    Retriever çalışmadan önce max_dense_similarity() çağrılır:
    Qdrant'tan top-k cosine skoru alınır. Eğer en yüksek skor
    rag_min_dense_similarity (varsayılan 0.38) altındaysa retriever
    boş döner → Grader "no" → WebSearch. Yabancı sorguların Qdrant'a
    girmesini yapısal olarak önler.
    Not: source_filter aktifken Dense Gate atlanır.

  Smart Reindex (fingerprint-based):
    Belge seti SHA-256 parmak izi .rag_cache/{collection}.fingerprint
    dosyasında tutulur. Parmak izi değişmemişse add_documents() atlanır.
    Aynı belgeyi ikinci kez yüklemek Qdrant'a yazma yapmaz.

  delete_by_source():
    on_chat_end'de session_uploads listesindeki dosyalar Qdrant'tan silinir.
    metadata.source_file filtresiyle hedefli temizlik yapılır.

5.2 Retrieval Stratejileri

  Strateji    | Yöntem                     | Uygun Senaryo
  ────────────┼────────────────────────────┼──────────────────────────
  hybrid      | Dense + BM25 fusion        | Genel (varsayılan)
  mmr         | Max Marginal Relevance     | "Neden/nasıl" — çeşitlilik
  threshold   | Skor eşiği filtresi        | Yüksek hassasiyet
  similarity  | Saf cosine similarity      | Belirli eşleşme

  auto_select_strategy() — Sorgu bazlı otomatik seçim:
    "kaç|süre" → hybrid
    "neden|nasıl" → mmr
    "hangi projelerde" → threshold

5.3 Dinamik K

  calculate_dynamic_k(question, base_k):
    "neden|nasıl|karşılaştır|fark" gibi karmaşıklık belirteçleri +4 ek doküman
    getirir. Temel k: 8, maksimum: 15.

5.4 Reranker — src/rag/reranker.py

  CrossEncoder (BAAI/bge-reranker-base) — sorgu-doküman çifti skoru:
    Batch size : 8
    TTL cache  : MD5(sorgu + doküman içerikleri + top_k) → 600s / 100 entry
    Immutability: Skor, Document nesnesini mutate etmeden yeni metadata ile
    yeni Document oluşturur.

  confidence_score:
    Rerank skoru mevcutsa birincil kaynak; yoksa Türkçe-aware term overlap.
    Grader'ın LLM kararı verip vermeyeceğini bu skor belirler.


────────────────────────────────────────────────────────────────────────────────
6. GRADER NODE — CRAG Implementasyonu
────────────────────────────────────────────────────────────────────────────────

Üç kademeli değerlendirme:

  Kademe A (LLM yok): confidence >= 0.75 → relevance = "yes"
  Kademe B (LLM yok): confidence < 0.15  → relevance = "no"
  Kademe C (LLM var): 0.15–0.75          → LLM grader (temp=0.0)

LLM grader artık üç-değerli JSON üretiyor:
  {"relevant": "yes"}
  {"relevant": "no", "reason": "irrelevant"}
  {"relevant": "no", "reason": "needs_live_data"}

_parse_grader_reason() regex ile reason alanını çıkarır; state'e yazar.

Hata yönetimi düzeltmesi:
  source_filter/session_uploads dalı LLM hatası → "yes" korundu (generator
  "belgede yok" der, makul).
  mid_conf dalı LLM hatası → önceki "yes" yerine "no"+"irrelevant" atandı.
  Böylece belirsiz durumda yetersiz bağlamla üretim yerine web fallback
  tercih edilir.

_grader_decision(state) kenar fonksiyonu:
  relevance == "yes"                              → "sufficient"
  source_filter + reason == "needs_live_data"     → "insufficient"
  source_filter + (diğer reason / yok)            → "sufficient"
  source_filter yok                               → "insufficient"

8. SEMANTİK CACHE — src/rag/semantic_cache.py
────────────────────────────────────────────────────────────────────────────────

Qdrant collection "semantic_cache" üzerinde çalışır.

Payload şeması:
  question   : normalize edilmiş sorgu (lowercase + whitespace)
  response   : tam metin yanıt
  timestamp  : Unix timestamp (TTL hesabı için)
  context_key: SHA-256(source_filter + sorted(session_uploads) +
               retrieval_strategy)[:16]

lookup(question, cache_ctx):
  Qdrant nearest-neighbor araması — filtreler:
    1. timestamp >= now - (ttl_hours * 3600)
    2. context_key == cache_ctx
  Eşik: semantic_cache_threshold (varsayılan 0.92 cosine)
  Cache hit → astream_agent() grafı çalıştırmaz.

store(question, response, cache_ctx):
  UUID ile yeni point upsert edilir.
  TTL veya context_key uyuşmazlığı eski kayıtları pasifleştirir.

Context key tasarım kararı:
  "Türkiye başkenti nedir?" sorusu hem genel chat'te hem de "Türkiye ekonomisi"
  adlı belgede sorulabilir. İkinci durumda cevap belgeden gelmeli. Aynı
  embedding yakın olsa da context_key farklı olduğu için cache çakışması olmaz.

Bypass koşulları (cache hiç çalışmaz):
  image_data doluysa → kesinlikle bypass
  input_type != "text" → bypass

12. FRONTEND & OTURUM YÖNETİMİ — src/main.py
────────────────────────────────────────────────────────────────────────────────

12.1 TTS/STT

  STT (Speech-to-Text):
    faster-whisper, CPU int8 quantization.
    on_audio_end: PCM buffer → WAV dönüşümü → transcribe → agent.
    _preload_whisper(): on_chat_start'ta asyncio.create_task() ile arka planda
    model yüklenir — ilk sesli sorguda gecikme olmaz.

  TTS (Text-to-Speech):
    edge-tts (Microsoft Azure Neural, API key gerektirmiyor).
    Dört ses: tr-TR-AhmetNeural, tr-TR-EmelNeural, en-US-AriaNeural,
    en-US-GuyNeural. "auto" modu Türkçe karakter tespiti ile seçer.

  _TtsStreamer — Paralel Sentez:
    feed(chunk): _MIN_FIRST_CHARS=150 karakter birikince cümle sınırında
    split eder, ilk grubun sentezini background'a başlatır.
    send_to(msg): İlk task await edilir, kalan metin sentezlenir, iki MP3
    byte-concatenate edilir → single seamless audio element.

  Temp dosya yönetimi (düzeltildi):
    _write_audio_tmp() artık session_upload_dir'e yazar (erişilebilirse).
    on_chat_end'deki shutil.rmtree() otomatik temizler.
    /tmp birikimi önlendi.

12.2 Konuşma Geçmişi — 3 Katmanlı Koruma

  _MAX_STORED_MESSAGES = 100    → RAM'de depolanan maksimum mesaj
  MAX_HISTORY_TURNS    = 20     → LLM context'ine gönderilen tur
  _MAX_HISTORY_CHARS   = 6000   → Token budget guard

  Oran: per-user context = LLM_CTX_SIZE / LLM_PARALLEL.
  16384 / 4 = 4096 token. System + RAG chunks ~2000-3000 token.
  Kalan ~1000-2000 token → ~4000-8000 karakter geçmiş.

  _build_lc_history(chat_history, summary):
    Önce tur limiti, sonra per-msg truncate (1500 char), sonra toplam
    karakter bütçesi kontrolü. Çift mesaj (user+assistant) birlikte
    kırpılır — geçmiş asistan mesajıyla başlamaz.

12.3 Uzun Süreli Bellek

  _SUMMARY_TRIGGER = 40    → Bu mesaj sayısından sonra özetleme tetiklenir
  _SUMMARY_KEEP_RECENT = 10 → Özetleme sonrası canlı tutulan son mesaj sayısı

  _summarize_and_compress_history():
    Eski mesajlar LLM'e "kısa ve bilgi yoğun özetle" promptuyla gönderilir.
    Özet thread metadata'ya (SQLite) kaydedilir.
    Chat resume'de yüklenerek SystemMessage olarak geçmişe eklenir.

12.4 Dosya İşleme Pipeline'ı

  Format          İşlem                            RAG'a girer?
  ─────────────────────────────────────────────────────────────────────────────
  PDF, DOCX       PyPDFLoader / UnstructuredWord    Evet
  TXT, MD         TextLoader                        Evet
  XLSX, CSV       UnstructuredExcel / CSV           Evet
  MP3/WAV/OGG     Whisper STT → TXT                 Evet (TXT olarak)
  PNG/JPG/WEBP    pending_vision_images listesi     Hayır (vision pipeline)
  URL (/url cmd)  WebBaseLoader scrape → TXT        Evet

  Limitler: 5 dosya / mesaj, 20 MB / dosya.
  Path traversal koruması: _track_session_upload() "/", "\", "." ile
  başlayan dosya adlarını reddeder.

12.5 Session Cleanup

  on_chat_end():
    1. get_hybrid_store().delete_by_source(session_uploads) → Qdrant temizliği
    2. shutil.rmtree(session_upload_dir) → dosya sistemi temizliği
    3. Session'daki TTS MP3 temp dosyaları da bu dizinde olduğundan otomatik
       silinir (önceden /tmp'ye yazılıp silinmiyordu).

13. DOCUMENT INGESTION — src/rag/ingest.py
────────────────────────────────────────────────────────────────────────────────

Üç katmanlı SRP tasarımı:

  DocumentLoader:
    LOADER_MAP {.pdf, .txt, .md, .docx, .xlsx, .csv} → OCP uyumlu pluggability.
    Yeni format: 1 satır ekleme yeterli.

  DocumentSplitter:
    RecursiveCharacterTextSplitter.
    PDF için özel separator sırası: "\n\n\n" → "\n\n" → "\n" → ". " → ...
    chunk_size=1200, chunk_overlap=200 (Gemma 4 E4B büyük context'i için).

  DocumentIngester:
    Loader + Splitter + vectorstore.add_documents() bileşkesi.

25. DİKKAT ÇEKEN TASARIM KARARLARI
────────────────────────────────────────────────────────────────────────────────

1. .replace() — not .format():
   RAG context { ve } içerebilir (JSON belgeleri, kod). Python str.format()
   çöker veya yanlış enjeksiyon yapar. Tüm prompt birleştirmelerinde
   .replace() kullanılıyor. Bilinçli karar.

2. original_question / question ayrımı:
   Rewriter retrieval için optimize ederken web search doğal dil ister.
   İki alan bu gergeyi zarif biçimde çözüyor.

3. ^ anchor routing pattern:
   _GENERAL_KNOWLEDGE_PATTERNS'ta "^... nedir?$" anchor. Uzun sorgularda
   "bağlantısı nedir?" gibi ifadenin false-positive oluşturmasını önler.

4. has_uploads ile _GENERAL_KNOWLEDGE bypass:
   Belge yüklüyken "transformer nedir?" → rag. Belgede geçen terimi
   soruyor olabilir. Uploads yokken → direct.

5. Grader confidence gate:
   BGE-M3 reranker skoru veya term overlap ile LLM öncesi eleme.
   Tipik sorgularda LLM grader çağrısından ~3s kaçınılır.

6. TTS temp dosya session dizini:
   Önceden /tmp'ye yazılan MP3'ler hiç silinmiyordu. Artık session_upload_dir
   altına yazılıyor; on_chat_end'deki shutil.rmtree() otomatik temizliyor.

7. Grader reason ayrımı (yeni):
   source_filter aktifken "no/irrelevant" → generator dürüstçe "belgede yok"
   der. "no/needs_live_data" → web fallback izin verilir. Önceki tasarım
   bu iki senaryoyu ayırt edemiyordu.

8. Cache context key:
   Aynı sorgunun farklı belge setlerinde farklı yanıtlar gerektirdiği
   durumda stale cache dönmesini önler. SHA-256 hash üretimi 0 ms.

9. asyncio.to_thread() kullanım yeri:
   Tüm CPU-bound veya sync IO işlemleri (embedding, reranking, SQLite,
   Qdrant, Whisper, TTS) to_thread ile event loop dışına alınmış.
   direct_response_node'daki llm.invoke() düzeltmesi bu kapsamda.
