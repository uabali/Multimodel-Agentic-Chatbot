2. KATMAN-KATMAN ANALİZ
────────────────────────────────────────────────────────────────────────────────

2.1 Frontend Katmanı — src/main.py
───────────────────────────────────

Chainlit üzerine kurulu frontend, aynı zamanda uygulamanın orkestrasyon giriş
noktasıdır.

Ses Hattı (STT/TTS):
- faster-whisper ile PCM → WAV dönüşümü sonrası transcription
  (asyncio.to_thread ile bloklamayan)
- _TtsStreamer sınıfı: LLM stream ederken PARALELDE ilk 150 karakterlik sesi
  sentezliyor, streaming bitince kalan ses ile MP3 concat ediyor.
  Tek audio element olarak gönderildiğinden seamless playback sağlıyor.
- Whisper ve Reranker modelleri on_chat_start'ta asyncio.create_task() ile
  background preload yapılıyor — ilk kullanımda gecikme yok.

Konuşma Geçmişi Yönetimi — 3 Katmanlı Koruma:
  _MAX_STORED_MESSAGES = 100    → RAM koruması (session dict'i)
  MAX_HISTORY_TURNS    = 20     → LLM context'e kaç tur gönderilir
  _MAX_HISTORY_CHARS   = 6000   → Token budget guard (4K per-user context)

Dosya İşleme Pipeline'ı:
  Format               İşlem
  ─────────────────────────────────────────────────────────────────
  PDF, DOCX, TXT, ...  ingest_file() → Qdrant
  PNG, JPG, WEBP       Base64 encode → pending_vision_images
  MP3, WAV, OGG        Whisper STT → TXT → ingest_file()
  URL (/url komutu)    WebBaseLoader → scrape → TXT → ingest_file()

Auth Güvenliği:
- PBKDF2-HMAC-SHA256, 210.000 iterasyon (OWASP 2024 önerisi ≥210K)
- hmac.compare_digest ile sabit-zamanlı karşılaştırma (timing attack önlemi)
- CHAINLIT_AUTH_SECRET yoksa rastgele oluşturulur, uyarı loglanır


2.2 Agent State — src/agent/state.py
──────────────────────────────────────

LangGraph TypedDict tanımı minimal ve temiz:

  class AgentState(TypedDict):
      messages: Annotated[Sequence[BaseMessage], add_messages]  # immutable accumulator
      original_question: str   # rewriter dokunmaz → web search bunu kullanır
      question: str            # rewriter overwrite eder
      vision_context: str      # VisionRAG çıktısı → Generator'a kaynak olarak girer
      session_uploads: list[str]  # follow-up RAG sorgularında kritik

original_question / question ayrımı önemli tasarım kararı: Rewriter sorguyu
vektör DB için optimize ederken web search'ün doğal dil sorgusunu koruması
gerekiyor. Bu ikili alan bunu zarif biçimde çözüyor.


2.3 Graph Topolojisi — src/agent/graph.py
──────────────────────────────────────────

  Router (keyword → 0ms | LLM fallback → ~200ms)
    ├─ vision_rag → Rewriter → Retriever → Grader → Generator
    ├─ vision_search → Generator (web + görsel, rewrite atlanır)
    ├─ vision → END (saf görsel analiz)
    ├─ rag → Rewriter → Retriever → Grader
    │         ├─ sufficient → Generator → END
    │         └─ insufficient → WebSearch → Generator → END  ← CRAG
    └─ direct → DirectResponse → END

Döngü Önlemi: web_search → generator kenarı grader'a geri dönmüyor.
Bu, sonsuz CRAG döngüsünü yapısal olarak (not runtime counter ile) engelliyor.


2.4 Router Node — 4 Kademeli Rota Kararı
──────────────────────────────────────────

  Kademe 0: image_data doluysa → "vision"   (0ms, LLM yok)
  Kademe 1: source_filter varsa → "rag"     (deterministic, 0ms)
  Kademe 2: keyword_route() eşleşirse       (0ms)
  Kademe 3: session_uploads + is_web_query() kombinasyonu
  Kademe 4: LLM fallback (Router LLM, max_tokens=64, ~200ms)

Router LLM çağrısı yalnızca belirsiz sorgularda yapılıyor. Tipik isteklerin
büyük çoğunluğu için sıfır LLM maliyeti.


2.5 Retriever Pipeline — src/rag/retriever.py + vectorstore.py
────────────────────────────────────────────────────────────────

Dense Gate: max_dense_similarity() ile Qdrant'a doğrudan sorgu atılarak en
yüksek cosine skoru alınır. Eğer < 0.45 ise retriever hiç çalıştırılmaz →
boş döner → Grader "no" → WebSearch. Yabancı sorguların RAG'a girmesini önlüyor.

Dinamik K: Soru karmaşıklığına göre k değeri base_k ± 4 aralığında ayarlanıyor.

4 Retrieval Stratejisi:
  hybrid     → Dense + BM25 sparse fusion (varsayılan)
  mmr        → "Neden/nasıl" — çeşitlilik öncelikli
  threshold  → Score eşiği altındakileri filtrele
  similarity → Saf cosine similarity

Reranker: BAAI/bge-reranker-base cross-encoder, _RerankerRegistry singleton
üzerinden lazy load.

Fingerprint-based Smart Reindex: SHA-256 ile belge seti parmak izi alınıyor;
değişmemişse Qdrant'a yeniden yazma atlanıyor.


2.6 Grader Node — CRAG Implementasyonu
────────────────────────────────────────

3 Kademe:
  Yüksek güven (≥0.75) → LLM atlanır → "yes"  (~3s kazanç)
  Düşük güven  (<0.15)  → LLM atlanır → "no"
  Orta güven (0.15–0.75) → LLM grader (temperature=0.0)

estimate_confidence() fonksiyonu rerank skoru varsa onu kullanır, yoksa
term overlap hesaplar. Turkish stopwords listesi mevcut.

Dosya bağlı sorgularda confidence atlanarak doğrudan LLM grader'a gidiyor —
yüklü dosyada gerçek zamanlı veri gerektiren sorgularda web search'ün devreye
girmesini sağlıyor.


2.7 Vision Sistemi — 3 Mod
────────────────────────────

  image_data + session_uploads  → vision_rag   (görsel analiz + RAG retrieval)
  image_data + is_web_query()   → vision_search (görsel analiz + web search)
  image_data (diğer)            → vision        (saf Gemma 4 multimodal)

select_vision_prompt() keyword matching ile (0ms) fatura/tablo/grafik/diyagram
için uzmanlaşmış promptlar seçiyor.
  "fatura" → JSON şeması çıkarımı
  "tablo"  → Markdown tablo formatı
  "grafik" → Veri nokta analizi
  "şema"   → Bileşen ve akış analizi


2.8 Çift-Geçişli PDF Ingestion
────────────────────────────────

  1. Geçiş: Metin (PyPDFLoader)
     documents = loader.load(pdf_path)
     chunks = splitter.split(documents)
     vectorstore.add_documents(chunks)

  2. Geçiş: Görsel (Gemma 4 Vision)
     pages = pdf2image.convert_from_path(pdf, dpi=150)

DPI=150: yeterli çözünürlük, aşırı büyük PNG'den kaçınma dengesi iyi kurulmuş.

6. DİKKAT ÇEKEN TASARIM KARARLARI
────────────────────────────────────────────────────────────────────────────────

Rewriter Hallucination Guard:
  Rewriter 250+ karakter üretirse veya cevap kalıpları içerirse orijinal soru
  korunuyor. Küçük model kullanıldığında halüsinasyonu önlüyor.

.format() yerine .replace():
  RAG context içinde { ve } karakterleri olabildiğinden (kod, JSON, PDF içeriği)
  prompt inject için Python str.format() kullanılmıyor — bilerek seçilmiş.

Web search'te original_question:
  Rewriter web sorgusu için optimize etmez; web search natural language query
  ile daha iyi sonuç veriyor.

llama.cpp backend guard:
  direct_response_node'da needs_mcp_tools() true olsa bile llama.cpp backend'de
  tool calling devre dışı — bu backend grammar-based JSON desteklemiyor.

Query Expansion (hava durumu):
  "5 günlük hava durumu" → "... 19 Nisan-23 Nisan 2026 günlük sıcaklık tahmin"
  Tavily'nin tarih aralığı bazlı snippet döndürmesini zorlayan ince teknik.

^-anchored routing patterns:
  _GENERAL_KNOWLEDGE_PATTERNS'ta ^ anchor kullanımı uzun sorularda false-positive
  önlüyor. "bağlantısı nedir?" → direct'e kaçırmıyor.

has_uploads ile _GENERAL_KNOWLEDGE_PATTERNS bypass:
  Belge yüklüyken "transformer nedir?" sorusu RAG'a gidiyor — belgedeki
  transformer mimarisini soruyor olabilir.

19. İYİLEŞTİRME ÖNERİLERİ
────────────────────────────────────────────────────────────────────────────────

1. main.py refactor (1179 satır):
   on_message handler'ı ayrı bir handlers/ modülüne taşınabilir.

2. Async PDF ingestion:
   VisualPageIngester._analyse_page() senkron. Büyük PDF'lerde UI donabiliyor.
   asyncio.gather() ile paralel sayfa analizi yapılabilir.

3. Session upload temizleme:
   on_chat_end'de upload dizini siliniyor ama Qdrant chunk'ları kalıyor.
   Uzun süreli kullanımda koleksiyon şişebilir. TTL veya manuel temizlik gerekli.

4. _graph singleton thread safety:
   if _graph is None → concurrent session'larda race condition riski.
   asyncio.Lock() veya başlangıçta eager init ile düzeltilebilir.

5. TTS temp dosya temizleme:
   tempfile.NamedTemporaryFile(delete=False) dosyaları silmiyor.
   Uzun çalışmada /tmp dolabilir. weakref veya atexit handler eklenebilir.

6. Reranker CPU bottleneck:
   BGE-Reranker-Base CPU'da çalışıyor. VRAM müsaitse GPU'ya taşınabilir.
   settings.reranker_device = "cuda" ile.

7. Qdrant GRPC:
   qdrant_prefer_grpc=True ayarlanmış ama port 6334 Docker'da açık mı
   kontrol edilmeli. HTTP/REST fallback var ama GRPC daha hızlı.
